# Start Here

Use this section when you want the shortest path to FDE competence. The goal is to create a basic map of the role and produce a few concrete artifacts.

## 20-Hour Starter Path

1. **Define the role and workflow.**
   - Read one FDE job posting and one customer story.
   - Output: one-page definition of what an FDE does.

2. **Pick and score one AI workflow.**
   - Use the use-case discovery resources.
   - Output: use-case scoring matrix with rejected alternatives.

3. **Design a thin architecture.**
   - Use AI architecture and LLM app engineering resources.
   - Output: architecture diagram and decision notes.

4. **Build or specify the thin slice.**
   - Use API, structured output, RAG, or tool-use resources.
   - Output: running prototype or implementation note.

5. **Evaluate and explain.**
   - Use eval, security, and rollout resources.
   - Output: eval report, risk review, demo narrative.
