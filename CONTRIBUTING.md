# Contributing

Thanks for helping improve Awesome FDE Resources.

## What To Contribute

Good contributions include:

- High-signal resources for a specific FDE task.
- Better categorization of existing links.
- Short explanations of why a resource matters.
- Playbooks, templates, or examples that help learners produce artifacts.

## Resource Acceptance Criteria

A resource should include:

- Link.
- Short title.
- Category.
- Why it matters for FDE work.
- Best-fit learner level: beginner, intermediate, advanced.
- What artifact it helps create.

Avoid:

- Link dumps.
- Paywalled-only resources with no public preview.
- Vendor pages that are mostly marketing.
- Generic AI content with no FDE-specific value.
- Duplicate resources unless they teach a materially different angle.

## Pull Request Format

Use this format:

```md
- [Resource title](https://example.com) - One sentence explaining why this helps an FDE task.
```

## Editorial Principles

- Prefer resources that help someone build, evaluate, deploy, or explain a system.
- Prefer practical, maintained, public materials.
- Keep categories task-based.
- When in doubt, include fewer links with better explanations.
