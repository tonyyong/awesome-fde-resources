# Evaluation Report Playbook

Use this when a prototype needs evidence that it is ready for pilot users.

## Build The Eval Set

- 10-30 realistic examples.
- Include easy, normal, ambiguous, stale, missing-data, unsafe, and adversarial cases.
- Record expected behavior and source documents when relevant.

## Score

Track:

- Task success.
- Groundedness or source correctness.
- Refusal quality.
- Retrieval hit rate.
- Structured output validity.
- Latency.
- Cost.
- Escalation or human-review rate.

## Report

Write:

- What was tested.
- What passed.
- What failed.
- Top failure modes.
- Fixes before pilot.
- Risks accepted for pilot.
- Owner and next eval date.
