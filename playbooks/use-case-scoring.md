# Use-Case Scoring Playbook

Use this when a team has many AI ideas and needs to choose the first deployment.

## Score Each Candidate

Rate each dimension from 1 to 5.

- Business value: clear workflow KPI, cost, revenue, risk, or speed impact.
- Feasibility: realistic data, integration path, and implementation scope.
- Data readiness: authoritative sources, freshness, structure, and access.
- Risk: security, compliance, brand, user trust, and operational downside.
- Adoption path: clear users, owners, incentives, and rollout motion.
- AI fit: model uncertainty is useful, not harmful; deterministic software alone is not enough.

## Reject Early When

- The workflow has no measurable owner.
- The data is unavailable, stale, or legally sensitive without controls.
- The first version requires high-risk autonomy.
- Users do not trust or want the workflow change.
- A rules engine, search page, or dashboard would solve the problem better.

## Output

Write one page:

- Recommended use case.
- Two rejected alternatives and why.
- Target users and current workflow.
- Success metric and baseline.
- Risks and controls.
- First pilot scope.
